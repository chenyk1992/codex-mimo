# Codebase Review: codex-mimo

## Summary

This review covers all 29 source files, 15 test files, and all documentation/config files in the codex-mimo project. The codebase is well-structured with clear separation of concerns, but has several critical bugs, security gaps, and documentation drift that need attention.

---

## Critical Issues (Must Fix)

### 1. `require()` in ESM module — runtime crash
**File:** `src/compose/runner.ts:270`

`buildReport` uses `const fs = require("node:fs")` inside an ESM-only project (`"type": "module"`). This will throw `ReferenceError: require is not defined` at runtime when any compose workflow produces a diff.

**Fix:** Add `import fs from "node:fs"` at the top of the file (it's already imported via other modules). Remove the inline `require()`.

### 2. `mimoReview` returns hardcoded empty findings
**File:** `src/codex/tools.ts:75`

The review tool always returns `findings: []` regardless of MiMoCode output. The `summary` field gets real data from `runAndCapture`, but structured findings are never extracted. This was flagged in the acceptance review as P1 and remains unresolved.

**Fix:** Parse MiMoCode output to extract findings, or remove the `findings` field from the return type if it's not yet supported.

### 3. `ciMode` parameter accepted but silently dropped
**File:** `src/cli/commands.ts:18`

`runImplement` accepts `ciMode?: boolean` but never passes it to `buildMimoRunArgs` or anywhere else. The CLI `--ci` flag is extracted in `main.ts` but has no effect on implementation commands.

**Fix:** Either pass `ciMode` through to the subprocess (e.g., as an env var or flag), or remove the parameter to avoid confusion.

## High-Severity Issues

### 4. JSON.parse in ACP parser has no try/catch
**File:** `src/mimo/acp-client.ts:27`

`JSON.parse(line)` in `JsonRpcLineParser.push()` will throw on malformed JSON, crashing the entire message handler. A single corrupted line kills the ACP session.

**Fix:** Wrap in try/catch, skip malformed lines (and optionally log a warning).

### 5. No request timeout in ACP client
**File:** `src/mimo/acp-client.ts:123`

`request()` creates a promise stored in `pending` with no timeout. If the server never responds, the promise hangs forever and leaks memory.

**Fix:** Add a configurable timeout (e.g., 30s default) that rejects the promise on expiry.

### 6. Session ID field name mismatch
**File:** `src/mimo/mimo-runner.ts:40`

The parser checks `m.sessionID` (capital "ID") but MiMoCode may return `sessionId` (lowercase "d"). This would cause `sessionId` to always be `null` if the field name differs.

**Fix:** Verify against actual MiMoCode output and handle both casings, or normalize.

### 7. `errors` array in mimo-runner is always empty
**File:** `src/mimo/mimo-runner.ts:72`

The `errors` field is initialized as `[]` and never populated. Error information from MiMoCode is lost.

**Fix:** Extract error events from JSONL output and populate the array.

### 8. `buildCommandLineFromParts` does not quote arguments
**File:** `src/mimo/acp-bridge.ts:314-316`

`[command, ...args].join(" ")` produces a flat string. Arguments with spaces are not quoted, causing incorrect shell parsing. Example: `["echo", "hello world"]` becomes `echo hello world` (3 args instead of 2).

**Fix:** Use proper shell quoting (e.g., wrap each arg in quotes if it contains spaces).

### 9. Symlink bypass in path containment check
**File:** `src/core/paths.ts:7`

`isPathInside` uses `path.resolve` which resolves `..` but does NOT follow symlinks. A symlink inside the workspace pointing outside would bypass the policy engine's containment check.

**Fix:** Use `fs.realpathSync` (with error handling for broken symlinks) before comparing paths.

## Medium-Severity Issues

### 10. Audit logger `rotate()` has async/sync race
**File:** `src/core/audit.ts:60-71`

`stream.end()` is async but immediately followed by `fs.renameSync`. The old stream may not have flushed before the rename, potentially losing the last few audit entries.

**Fix:** Use `stream.end()` callback or `stream.close()` with a callback before renaming.

### 11. Terminal stdout/stderr buffers grow unbounded
**File:** `src/core/terminal.ts:78-83`

`terminal.stdout += data.toString()` appends without limit. `TerminalCreateParams` has an `outputByteLimit` field but it's never enforced.

**Fix:** Enforce `outputByteLimit` by truncating or discarding data beyond the limit.

### 12. `cleanup()` counts rotated files incorrectly
**File:** `src/core/audit.ts:50-57`

`files.length >= this.maxFiles` checks rotated files but doesn't account for the current `audit.jsonl`. After rotation, total files could be `maxFiles + 1`.

**Fix:** Account for the current file in the count: `files.length >= this.maxFiles - 1`.

### 13. `JSON.parse` unsafe cast in config loading
**File:** `src/core/config.ts:34`

`JSON.parse(raw) as ConfigFile` is an unsafe cast. No schema validation means malformed config causes runtime errors later.

**Fix:** Use Zod to validate the config file (Zod is already a dependency).

### 14. Sessions file has no validation on load
**File:** `src/core/sessions.ts:54-60`

`JSON.parse(raw) as SessionEntry[]` is an unsafe cast. Corrupted sessions file causes runtime errors.

**Fix:** Validate with Zod or at minimum check that the parsed value is an array.

### 15. Sessions grow without pruning
**File:** `src/core/sessions.ts`

Sessions are never pruned. Over time, `sessions.json` grows without bound.

**Fix:** Add a max session count or TTL-based pruning.
